# 토스페이먼츠 가맹가입 심사 회신·작업 준비

작성일: 2026-06-22

이 문서는 토스페이먼츠 계약/카드사 심사 메일에 바로 회신할 수 있는 초안과, 자동화 에이전트 `젠스파크크루`가 결제경로 PPT 및 홈페이지 점검을 이어서 수행하기 위한 작업 체크리스트입니다.

## 0. 현재 코드 기준 확인된 사항

- 공개 상품/서비스 노출: 홈 `/`의 `#products` 섹션에서 수강권 상품 카드 노출
- 회원 구매 화면: 로그인 후 `/app` → `수강권 구매` 탭에서 상품 상세/이용기간/환불정책 확인 후 구매
- 환불정책: `/refund`
- 약관/개인정보처리방침: `/terms`, `/privacy`
- 사업자정보 footer: `BusinessFooter`에서 하단 노출
- 직접 결제 연동: Toss SDK + 자체 API 연동
  - checkout: `/api/payments/checkout`
  - success redirect: `/payments/success`
  - fail redirect: `/payments/fail`
  - server confirm: `/api/payments/confirm`
  - webhook: `/api/payments/webhook`
- 운영 도메인 후보: `https://runclub-manager.onrender.com`
  - `render.yaml` 기준 Render 기본 외부 URL입니다. 실제 토스 신청서에 넣은 도메인이 따로 있으면 아래 회신 URL을 그 도메인으로 교체하세요.

## 3. 젠스파크크루 작업 지시서

### 목표

토스페이먼츠 가맹가입/카드사 심사가 반려되지 않도록 아래 산출물을 준비합니다.

1. 토스 메일 회신값 최종 확정
2. 홈페이지 준비사항 점검
3. 결제경로 PPT 제작
4. 테스트계정 로그인/결제 플로우 확인

### 작업 순서

#### A. URL/화면 검증

1. 운영 URL 접속
   - `https://runclub-manager.onrender.com/`
   - 상품 섹션 `#products`가 보이는지 확인
2. 환불정책 접속
   - `https://runclub-manager.onrender.com/refund`
3. 약관/개인정보처리방침 접속
   - `https://runclub-manager.onrender.com/terms`
   - `https://runclub-manager.onrender.com/privacy`
4. 하단 사업자정보 확인
   - 상호명, 대표자명, 사업자등록번호, 주소, 전화번호, 이메일, 통신판매업 신고번호가 보이는지 캡처
5. 공개 상품 카드 확인
   - 상품명, 가격, 이용기간이 보이는지 캡처

#### B. 테스트계정 준비

1. 관리자 계정으로 접속
2. 토스 심사용 일반 회원 생성 또는 기존 테스트 회원 활성화
3. 임시 비밀번호 발급 후 `must_change_password`가 걸리면 실제 비밀번호 변경까지 완료
4. 심사관에게 줄 계정으로 직접 로그인 테스트
5. `/app` → 수강권 구매 화면 접근 확인

#### C. 결제경로 PPT 구성

토스 예시파일과 동일한 구성요소를 맞추되, 최소 아래 슬라이드를 포함합니다.

1. **홈페이지 메인/상품 노출**
   - URL 주소창 포함
   - 상품/서비스 카드가 보이게 캡처
2. **회원가입 또는 로그인 화면**
   - 비회원 구매 불가 및 로그인 필요 흐름 설명
3. **회원 로그인 후 수강권 구매 화면**
   - `/app` 화면
   - `수강권 구매` 탭/상품 목록 캡처
4. **상품 상세 화면**
   - 상품명, 가격, 이용기간, 횟수, 환불정책, `결제하기` 버튼이 보이게 캡처
5. **토스 결제창**
   - `결제하기` 클릭 후 토스페이먼츠 결제창 캡처
   - 테스트 결제 모드라면 테스트 모드임을 명시
6. **결제 성공 화면**
   - `/payments/success` 화면 캡처
   - “결제가 완료되어 수강권이 발급되었습니다” 메시지 확인
7. **마이페이지/수강권 발급 확인**
   - 결제 후 내 수강권에 발급된 내역 캡처
8. **환불정책 페이지**
   - `/refund` 캡처
9. **사업자정보 하단**
   - footer 사업자정보 캡처

각 슬라이드 하단에는 “고객 동선: 접속 → 로그인 → 상품 선택 → 상세 확인 → 결제창 → 결제 완료 → 수강권 발급” 순서를 짧게 적습니다.

#### D. 결제 테스트 주의사항

- 운영 키가 아직 없으면 Toss 테스트 키로 진행합니다.
- 테스트 카드 정보는 화면 배너 기준:
  - 카드번호: `4330-1234-1234-1234`
  - 유효기간: `12/30`
  - CVC: `123`
  - 비밀번호: `00`
- 실제 금액 결제가 일어나지 않는 테스트 모드인지 확인합니다.
- 결제 성공 후 서버 승인(`/api/payments/confirm`)과 수강권 발급까지 확인합니다.

#### E. 최종 산출물

1. 토스 회신 메일 최종본
2. 결제경로 PPT 파일
3. 테스트계정 ID/PW
4. 점검 체크 결과
   - 상품 URL 정상
   - 환불정책 URL 정상
   - 사업자정보 footer 정상
   - 결제창 정상
   - 결제 성공/수강권 발급 정상
