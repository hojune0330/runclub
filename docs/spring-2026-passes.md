## 1. 확정 규칙 (오너 확정 — 재논의 금지)

이용권의 **이용시작/이용종료**는 손으로 적지 않고 아래 규칙으로 **자동 계산**합니다.
(`src/lib/pass-term.ts` 의 `computePassTerm()` 단일 함수가 책임)

| 구분 | 조건 | 이용시작 | 이용종료 |
|------|------|----------|----------|
| **4월 결제 (개강 대기 특수)** | 결제 월이 개강 월(5월)보다 이름 | **개강일 5/6 고정** | 5/6 + (개월수 × 30일) |
| **5월 이후 결제 (안내문 규칙)** | 그 외 | **결제일 = 시작일** | 결제일 + (개월수 × 30일) |

- **개강일**: `2026-05-06` (상수 `OPENING_DATE_2026`)
- **1개월 = +30일** 고정(달력 월말 보정 없음). 운영 장부가 30일 기준이라 그대로 맞춤.
- **정연경**: 결제일(6/2) 기준 그대로 진행 (장부의 6/3 시작 표기는 미사용 — 오너 확정).

### 4월 개강대기 종료일 (시작 5/6 고정)
- 1개월(+30일) → **6/5**
- 2개월(+60일) → **7/5**
- 3개월(+90일) → **8/4**

## 3. 시스템 반영 현황 (왜 자동 import가 아닌가)

이 장부는 **member_passes(이용권 발급)** 데이터입니다. 현재 시스템 구조상:

1. `member_passes.member_id` 는 **FK** — 먼저 `members` 에 회원이 있어야 발급 가능.
2. Google Sheets 가져오기 로직은 **회원 메타데이터(J~O)만** 대상이라 이용권 발급은 범위 밖.
3. 이용권은 관리자 웹의 **직접 지급**(`pass_grant_records`) 또는 결제 플로우로 생성.

→ 그래서 "표 그대로 자동 반영"은 안 되고, 대신 **검증된 fixture + import
스크립트**로 회원이 DB에 존재할 때 안전하게 일괄 발급하도록 만들었습니다.

| 자산 | 경로 | 역할 |
|------|------|------|
| 기간 규칙 | `src/lib/pass-term.ts` | 결제일·개월 → 시작/종료 단일 계산 |
| 장부 정본 | `src/lib/spring-2026-passes.ts` | 40건 raw(이름·결제일·개월·금액) + 파생 |
| 골든 테스트 | `scripts/verify-spring-2026-passes.mjs` | 코드↔확정표 일치 검증(DB 불필요) |
| 일괄 발급 | `scripts/import-spring-2026-passes.mjs` | 이름→회원 매칭 후 발급(dry-run 기본) |

## 4. 운영 매뉴얼

### 4.1 표/코드 일치 검증 (DB 없이 가능)
```bash
npm run passes:verify-spring
# → "✅ 40개 행이 확정 장부와 모두 일치합니다." 가 나오면 정상
```

### 4.2 일괄 발급 dry-run (무엇이 발급/보류될지 확인)
```bash
# DATABASE_URL 설정 후
npm run passes:import-spring
# → 발급 가능 N건 / 보류 M건(unmatched·ambiguous·already issued) 리포트만 출력
```

### 4.3 동명이인 해소 (--map)
보류된 `ambiguous` 행은 매핑 파일로 휴대폰을 지정합니다.
키는 `"이름#장부순서(0-based)"` 또는 `"이름"`, 값은 휴대폰 또는 memberId.
```json
{
  "유명훈#7": "010-1111-1111",
  "유명훈#10": "010-2222-2222",
  "김준택#4": "010-3333-3333",
  "김준택#27": "010-4444-4444",
  "서보경#28": "010-5555-5555",
  "서보경#35": "010-6666-6666"
}
```
```bash
npx tsx scripts/import-spring-2026-passes.mjs --map map.json
```

### 4.4 실제 발급
```bash
npm run passes:import-spring -- --apply
# 또는 매핑과 함께
npx tsx scripts/import-spring-2026-passes.mjs --map map.json --apply
```
- 트랜잭션으로 전체 발급, 실패 시 롤백.
- **멱등성**: 같은 (회원·상품·시작일·발급일)이 이미 있으면 건너뜀 → 두 번 돌려도 안전.
- 만료일이 오늘보다 과거면 `status='expired'`, 아니면 `'active'` 로 발급.
- 결제 상태는 `paid` / `cash` 로 기록(장부 = 현금/계좌 수금 가정. 다르면 스크립트에서 조정).

## 5. 장부에서 새 회원/결제를 추가할 때

1. `src/lib/spring-2026-passes.ts` 의 `SPRING_2026_RAW` 에 한 줄 추가
   (이름·결제일·개월·금액만 — 시작/종료는 자동 계산).
2. `npm run passes:verify-spring` 의 기대값(`scripts/verify-spring-2026-passes.mjs`)도 같이 갱신.
3. dry-run → (필요 시 --map) → --apply 순으로 발급.

> 규칙이 바뀌면 `src/lib/pass-term.ts` **한 곳만** 고치면 모든 화면·스크립트에 반영됩니다.
