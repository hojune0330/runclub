# RunClub Google Sheets 매니저 가이드

> **누구를 위한 문서인가**: 회원/수강권/출석 데이터를 시트에서 직접 다루는
> 운영 매니저용 가이드입니다. 코드 수정 권한이 없어도 시트 사용만으로
> 일상적인 회원 관리·메모 작성·엑셀 백업이 가능합니다.

---

## 1. 시트 구성 한눈에 보기

| 탭 | 설명 | 동기화 방식 |
|------|------|------------|
| **Members** | 회원 마스터 (이름·연락처·이메일·메모 등) | 단방향 미러 |
| **Passes** | 수강권 발급/잔여/상태 | 단방향 미러 |
| **Attendance** | 출석 이력 (영구 누적) | append-only |
| **Sessions** | 세션 정보 | 단방향 미러 |

**단방향 미러**: 웹/앱에서 회원이 변경하면 시트에 자동 반영. 시트에서
직접 수정해도 DB로 거꾸로 가지 않습니다.

---

## 2. 컬럼이 두 종류로 나뉘어 있어요

각 탭은 **DB 영역**과 **매니저 메모 영역**으로 구분돼 있습니다.

### Members 탭

| 컬럼 | 항목 | 누가 쓰는가 |
|------|------|-------------|
| A~I | 회원ID, 이름, 연락처, 이메일, 권한, 가입일, 활성여부, 시스템메모, 최종동기화 | 🔒 시스템 자동 (수정 금지) |
| J | 매니저 메모 | ✏️ 매니저 자유 입력 |
| K | 태그 (드롭다운: VIP/이탈주의/신규/휴면/기타) | ✏️ 매니저 |
| L | 회원 등급 (드롭다운: 일반/우수/VIP/블랙) | ✏️ 매니저 |
| M | 유입 경로 (드롭다운: 지인추천/인스타그램/검색/광고/직접방문/기타) | ✏️ 매니저 |
| N | 다음 컨택 예정일 | ✏️ 매니저 (날짜) |
| O | 담당 매니저 | ✏️ 매니저 |

### Passes 탭

| 컬럼 | 항목 | 누가 쓰는가 |
|------|------|-------------|
| A~N | 수강권 정보 일체 | 🔒 시스템 자동 |
| O | 매니저 메모 | ✏️ 매니저 (환불 사유, 연장 약속 등) |

### Sessions 탭

| 컬럼 | 항목 | 누가 쓰는가 |
|------|------|-------------|
| A~M | 세션 정보 일체 | 🔒 시스템 자동 |
| N | 매니저 코멘트 | ✏️ 매니저 (예: "비 예보로 단축 운영") |

### Attendance 탭

전체 컬럼이 **append-only 이력**입니다. 손으로 수정해도 다음 동기화에
영향이 없지만, **임의로 행을 삭제하면 출석 기록이 사라지므로 금지**.

## 3. 매일 사용하는 작업 흐름

### 3.1 신규 가입자 확인
1. **Members** 탭 → F열(가입일) 기준 내림차순 정렬
2. 가장 위쪽이 최근 가입자
3. M열(유입 경로), L열(회원 등급), J열(매니저 메모) 채우기

### 3.2 회원 상담/연락 메모
1. 해당 회원 행을 검색 (Ctrl/Cmd+F)
2. **J열(매니저 메모)** 에 자유롭게 기록
3. 다음 컨택할 날짜를 **N열** 에 입력 → Apps Script 알림 자동 발송

### 3.3 이탈주의 회원 마킹
1. **K열 태그**에서 드롭다운 → "이탈주의" 선택
2. 매주 자동 리포트(`weeklyChurnReport`)가 이 라벨 기준으로 추적

### 3.4 환불/일시정지 처리
1. **Passes** 탭에서 해당 수강권 행 검색
2. K열(상태)이 자동 갱신되어 있는지 확인
3. **O열 매니저 메모**에 환불 사유, 환불일, 처리자 기록

---

## 4. 엑셀 다운로드 (오프라인 백업)

1. 시트 상단 메뉴 → **파일 → 다운로드 → Microsoft Excel(.xlsx)**
2. 모든 탭이 하나의 .xlsx 파일로 묶여서 다운로드
3. 파일명에 날짜 포함 권장: `runclub_2026-05-06.xlsx`

> 정기 백업이 필요하면 매주 월요일 오전에 다운로드만 해도 충분합니다.

## 5. 자주 묻는 질문 (FAQ)

### Q. 시트에서 회원 이름을 바꾸면 앱에도 반영되나요?
**아닙니다.** A~I 컬럼은 단방향 미러라 시트에서 수정해도 DB로 가지 않고,
다음 동기화에서 원복됩니다. 회원 정보 수정은 반드시 **관리자 웹/앱**에서
진행하세요.

### Q. 매니저 메모(J~O 등)를 잘못 지웠어요.
시트의 **수정 기록(파일 → 버전 기록)** 으로 복구할 수 있습니다. 시트는
자동으로 모든 편집을 기록합니다.

### Q. 시트가 보이지 않습니다 / 권한 거부 에러가 떠요.
관리자에게 시트 **편집자 권한** 부여를 요청하세요. 새 매니저 합류 시에도
같은 방식으로 인계됩니다.

### Q. 회원 행 일부 컬럼이 잠겨 있어 수정이 안 돼요.
정상입니다. A~I 컬럼은 시스템 보호 영역이라 매니저는 수정 불가입니다.
J 이후의 컬럼만 자유롭게 입력 가능합니다.

### Q. 신규 회원이 시트에 안 나타나요.
1~2초 정도 지연은 정상입니다. 1분 이상 누락되면 관리자에게 문의하세요
(`sheet_sync_queue`에 큐잉됐을 가능성이 있고, 1분 단위 cron이 자동
재시도합니다).

## 6. 절대 하지 말아야 할 것

- ❌ 시트에서 **A~I (Members), A~N (Passes), A~M (Sessions)** 컬럼을
  손으로 편집 (보호되어 있어 시도 자체가 차단됨)
- ❌ Attendance 탭에서 행 삭제 (출석 이력 손실)
- ❌ 시트를 **외부에 공개 공유** (개인정보 포함, 절대 금지)
- ❌ 시트 ID나 Service Account JSON을 매니저들끼리 공유

---

## 7. 관리자 인계 시 체크리스트

새로운 매니저에게 운영을 인계할 때:

1. ✅ Google 시트 **편집자**로 추가 (시트 우측 상단 [공유])
2. ✅ 본 가이드 문서 링크 공유
3. ✅ 자동화 알림 수신 메일이 있다면 `automation-samples.gs`의
   `NOTIFY_TO` 값 변경
4. ✅ 기존 매니저 권한 회수 (필요 시)
5. ✅ 첫 1주일 동안 함께 작업하면서 사용법 안내

## 8. 트러블슈팅

| 증상 | 원인 후보 | 조치 |
|------|----------|------|
| 신규 가입이 시트에 5분 이상 누락 | Sheets API 일시 장애 | 자동 cron 재시도 대기 또는 관리자 문의 |
| 매니저 메모가 갑자기 사라짐 | 실수 삭제 | 파일 → 버전 기록으로 복구 |
| 드롭다운 선택지가 부족 | 신규 옵션 필요 | 관리자에게 요청 → `setup.gs` 또는 `sheet-init.mjs` 수정 |
| 행이 너무 많아서 느림 | 누적 데이터 증가 | Attendance 탭은 분기별로 별도 시트 분리 검토 |

## 9. 관련 문서

- 시트 자동 초기화 스크립트: `scripts/sheet-init.mjs` (관리자 전용)
- Apps Script 보호 재적용: `docs/apps-script/setup.gs`
- Apps Script 자동화 샘플: `docs/apps-script/automation-samples.gs`
- 환경변수: `.env.example` 의 `SHEET_SYNC_*` 섹션
